# Layout Convention naming

In Rasengan.js, to create a `layout`, you have to create a file that follow the following name pattern: 

```txt title="Layout" showLineNumbers
[name].layout.js
```

A `layout` is UI that is shared between routes.

## Example 

<Tabs tabs={[{ title: 'TypeScript' }, { title: 'JavaScript' }]} activeIndex={0}>
  <Tabs.Item>
    ```tsx title="app.layout.tsx" showLineNumbers 
    import { LayoutComponent, Outlet } from "rasengan";

    /*
     * This AppLayout is located inside the app.layout.tsx file
     */
    const AppLayout: LayoutComponent = () => {
      return (
        <div>
          <h1>App Layout</h1>

          <Outlet />
        </div>
      )
    }
    
    AppLayout.path = "/";

    export default AppLayout;
    ```
  </Tabs.Item>
  <Tabs.Item>
    ```jsx title="app.layout.jsx" showLineNumbers 
    import { Outlet } from "rasengan";

    /*
     * This AppLayout is located inside the app.layout.tsx file
     */
    const AppLayout = () => {
      return (
        <div>
          <h1>App Layout</h1>

          <Outlet />
        </div>
      )
    }
    
    AppLayout.path = "/";

    export default AppLayout;
    ```
  </Tabs.Item>
</Tabs>

Based on the convention, we have juste replace the `[name]` by `app` in the convention name and we got `app.layout.jsx`.

{/* <AlertBox>
Following the convention, is not required but, keeping that convention will help you recognizing the purpose of each files you will have.
</AlertBox> */}
